# 이슈 및 개선사항 (선택사항)

## 이슈

- **빌드 용량**: 처음 빌드했을 때 산출물이 700KB를 넘어, Vite 기본 기준(500KB)을 초과해 빌드 경고가 발생함. 대응으로 각 라이브러리 그룹별로 청크를 분리하는 방법을 적용함.

## 향후 개선 계획

- **채팅 단위로 로드**: 채팅 단위로 필요할 때만 불러오도록 구조 변경 필요 (현재는 전체 데이터를 로드)
- **가상 리스트 도입**: 채팅 내 대규모 메시지 성능 최적화
- **도메인 특화**: 도메인 특화 지식 기반 RAG 기능
- **Function call 도입**: 현재는 채팅 기반 기능이지만, function call을 도입해 채팅·이미지 등 다양한 기능을 채팅 내에서 제공할 수 있도록 확장 필요.
- **커스텀 에이전트, 워크플로우**: MCP 같은 도구들을 연동하고 워크플로우를 생성할 수 있는 기능을 만들어, 정말 업무에 도움을 주는 **커스텀 가능한 에이전트**를 제공. 만든 에이전트를 공유해서 사용할 수 있게 하고, **오픈 에이전트** 형태로 에이전트의 워크플로우, 프롬프트를 수정해 활용할 수 있도록 하는 기능.
